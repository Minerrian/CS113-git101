การ Branch Merge Conflict
คำสั่งเกี่ยวกับbranch
1.การสร้างbrabch - git branch ชื่อที่จะตั้ง
2.การตรวจว่ามีbranchอะไรบ้าง  - git branch
3. การเปลี่ยน branch - git checkout ชื่อbranchที่อยากจะย้ายไป
4.การสร้างbranchแล้วให้สลับไปใช้branchนั้นเลย git checkout -b ชื่อbranchที่ต้องการสร้าง

การ Mergeแบบไม่มีconflict
1.เราจะมีbranch 2 อันและเพิ่มข้อมูลเข้าไปในทั้ง2อัน ด้วยการpushข้อมูลที่เราต้องการให้ส่งขึ้นไป
2.จากนั้นเราต้องเปลี่ยนไปที่ branch ปลายทางก่อน
3.แล้วจึงค่อยใช้คำสั่ง git merge branchที่เราจะให้ไปอยู่ในbranchปลายทาง
4.เมื่อไม่มี conflict จะทำให้ git สร้าง commit ของการ merge นั้นไปเลย ดังนั้น เราจะต้องใส่ commit message 
ด้วย vi ถ้าใช้ default merge commit message ให้กด ESC แล้วพิมพ์ :wq! แล้วกด Enter
5.เมื่อเราmergeไปแล้วเราก็จะเห็นว่าไฟล์ของทั้ง2ที่เราpushขึ้นไปในแต่ละbranchนั้นจะอยู่ในbranchปลายทางแล้ว
6.จากนั้นpushอีกครั้งไฟล์ทั้งหมดก็จะขึ้นในgithubของเราแล้ว

Pullแบบไม่มีconflict
1.เราต้องclone git reponsitory มาไว้อีกเครื่องหรือ ไดเกทอรี่ใหม่
2.สมมติเราสร้างไฟล์ใหม่ในนี้ แล้วpushแล้ว
3.กลับมาที่ไดเรกทอรี่เก่าหรือเครื่องเก่า แล้วพิมพ์ ls -l จากนั้นทำการพิมพ์คำสั่ง git pull ชื่อremote ชื่อbranch จากนั้น ls -l อีกครั้ง
ในไดเรกทอรี่เก่าเราก็จะมีไฟล์แบบเดียวกับไดเรกทอรี่2แล้ว

Merge แบบมีconflict
Conflict คือ ความขัดแย้งที่เกิดจากการปรับแก้ไขข้อมูลในไฟล์เดิม บรรทัดเดิม ที่อยู่ต่าง Branchกัน
เมื่อ merge แล้ว git ตัดสินใจไม่ได้ว่าจะใช้ข้อมูลจาก branch ใด
1.เมื่อเราmergeแล้วมันเกิดConflict ถ้าเรามีeditor ที่มีตัวช่วยเราก็จะสามารถเลือกได้เลยว่าเราจะใช้ข้อมูลจากbranchไหน
แต่ถ้าไม่มีตัสช่วย เราจึงต้องทำการ เพิ่มหรือแก้หรือลบเอง
2.จากนั้นpushขึ้นไปในgithub

Pull แบบมีconflict
เมื่อมีการเปลี่ยนแปลงข้อความแล้ว เราต้องทำการpullก่อน แต่เมื่อเราpull คราวนี้มันจะเกิดconflictขึ้น 
ให้แก้conflictจากนั้นaddไฟล์อีกรอบแล้วcommit จากนั้นก็pushได้เลย
